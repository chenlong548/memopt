# memopt 使用完全指南

memopt 是一个高性能内存优化工具包，专注于解决现代计算环境中的内存相关挑战，包含 8 个核心模块，提供内存映射、数据压缩、流式处理、内存优化、惰性计算、稀疏数组、内存监控和缓冲区管理等完整功能。

## 适用技术场景

### 1. 大模型训练与推理
- **内存映射**：高效处理大型模型权重文件和数据集
- **数据压缩**：压缩 KV Cache 和模型权重，减少内存占用
- **稀疏数组**：优化稀疏矩阵运算，加速注意力机制
- **内存监控**：实时监控内存使用，避免 OOM 错误

### 2. 大数据处理
- **流式处理**：处理海量数据流，支持乱序数据和窗口操作
- **内存映射**：直接映射大型数据集，避免数据拷贝
- **缓冲区管理**：高效管理 I/O 缓冲区，提高数据吞吐量
- **数据压缩**：压缩中间结果，减少网络传输和存储开销

### 3. 实时系统
- **内存优化**：使用 TLSF 分配器，保证内存分配的实时性
- **缓冲区管理**：无锁队列和环形缓冲区，减少线程同步开销
- **惰性计算**：延迟计算，避免不必要的计算开销
- **内存监控**：低开销监控，及时发现内存问题

### 4. 高性能计算 (HPC)
- **NUMA 优化**：充分利用多 NUMA 节点，提高内存访问性能
- **大页支持**：使用 2MB/1GB 大页，减少 TLB 缓存 misses
- **稀疏数组**：GPU 加速的稀疏矩阵运算
- **内存监控**：NUMA 感知的内存使用分析

### 5. 嵌入式系统
- **内存优化**：高效内存分配，减少内存碎片
- **数据压缩**：压缩存储数据，减少内存占用
- **惰性计算**：按需计算，节省系统资源
- **内存监控**：轻量级监控，不影响系统性能

## 1. mem_mapper - 内存映射工具

内存映射工具提供高性能的内存映射功能，支持 NUMA 感知、大页优化、智能预取等高级特性。

### 1.1 核心类

#### MemoryMapper

内存映射器主类，整合所有功能模块。

**构造函数：**
```python
from mem_mapper import MemoryMapper, MapperConfig

config = MapperConfig(
    use_numa=True,           # 是否使用 NUMA 优化
    use_huge_pages=True,     # 是否使用大页
    use_prefetch=True,        # 是否使用预取
    numa_policy=NUMAPolicy.DEFAULT,  # NUMA 策略
    numa_node=-1,             # 默认 NUMA 节点（-1 表示自动）
    huge_page_size=0,         # 大页大小（0 表示自动选择）
    min_size_for_huge_pages=2*1024*1024,  # 使用大页的最小大小
    prefetch_strategy=PrefetchStrategyType.SEQUENTIAL,  # 预取策略
    prefetch_window=16,      # 预取窗口大小
    cleanup_delay=60.0,      # 清理延迟（秒）
    idle_threshold=300.0,    # 空闲阈值（秒）
    max_mappings=10000,      # 最大映射数量
    enable_stats=True,       # 是否启用统计
    async_prefetch=False     # 是否异步预取
)

mapper = MemoryMapper(config)
```

**主要方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `map_file()` | path, mode, offset, size, numa_node, use_huge_pages, prefetch | MappedRegion | 映射文件到内存 |
| `unmap()` | region | None | 解除映射 |
| `advise()` | region, advice | None | 提供访问模式建议 |
| `sync()` | region, async_mode | None | 同步映射到文件 |
| `lock()` | region | None | 锁定映射（防止换出） |
| `unlock()` | region | None | 解锁映射 |
| `get_stats()` | - | Dict | 获取统计信息 |
| `get_region()` | region_id | MappedRegion | 获取指定映射区域 |
| `get_all_regions()` | - | List[MappedRegion] | 获取所有映射区域 |
| `cleanup()` | - | None | 清理所有映射 |

**示例：**
```python
from mem_mapper import MemoryMapper, MapperConfig

# 创建映射器
config = MapperConfig(use_numa=True, use_huge_pages=True)
mapper = MemoryMapper(config)

# 映射文件
region = mapper.map_file(
    path='/path/to/file.bin',
    mode='readonly',      # readonly | readwrite | writecopy
    offset=0,             # 文件偏移
    size=0,               # 映射大小（0 表示整个文件）
    numa_node=-1,         # NUMA 节点（-1 表示自动）
    use_huge_pages=None,  # None 表示使用配置默认值
    prefetch=True         # 是否预取
)

# 读取数据
data = region.address[:region.size]

# 提供访问建议
mapper.advise(region, 'sequential')  # sequential | random | willneed | dontneed

# 同步脏数据
mapper.sync(region, async_mode=False)

# 解锁（如果之前锁定了）
mapper.unlock(region)

# 解除映射
mapper.unmap(region)

# 清理所有映射
mapper.cleanup()
```

**上下文管理器用法：**
```python
with MemoryMapper() as mapper:
    region = mapper.map_file('/path/to/file.bin', mode='readonly')
    data = region.address[:region.size]
# 自动清理
```

## 2. data_compressor - 数据压缩工具

数据压缩工具提供高性能的数据压缩功能，支持多种压缩算法和自适应选择。

### 2.1 核心类

#### DataCompressor

数据压缩器主类，提供自适应压缩功能。

**构造函数：**
```python
from data_compressor import DataCompressor, CompressionConfig, CompressionAlgorithm, CompressionLevel

config = CompressionConfig(
    algorithm=CompressionAlgorithm.ZSTD,  # 压缩算法
    level=CompressionLevel.BALANCED,      # 压缩级别
    enable_stats=True,                   # 启用统计
    verify_compression=True,             # 验证压缩结果
    streaming=False                       # 流式压缩模式
)

compressor = DataCompressor(config)
```

**主要方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `compress()` | data, config | CompressedData | 压缩数据 |
| `decompress()` | compressed | bytes | 解压数据 |
| `analyze()` | data | Dict | 分析数据特征 |
| `benchmark()` | data, algorithms | Dict | 基准测试算法 |
| `get_capabilities()` | - | Dict | 获取算法能力 |
| `get_stats_summary()` | - | Dict | 获取统计摘要 |

**示例：**
```python
from data_compressor import DataCompressor, CompressionConfig, CompressionAlgorithm, CompressionLevel

# 创建配置
config = CompressionConfig(
    algorithm=CompressionAlgorithm.ZSTD,
    level=CompressionLevel.FAST
)

# 创建压缩器
compressor = DataCompressor(config)

# 压缩数据
data = b"Hello, World! " * 1000
compressed = compressor.compress(data)

print(f"原始大小: {compressed.original_size}")
print(f"压缩后: {compressed.compressed_size}")
print(f"压缩比: {compressed.stats.compression_ratio:.2f}x")

# 解压数据
decompressed = compressor.decompress(compressed)
assert data == decompressed
```

## 3. stream_processor - 流式处理工具

流式处理工具提供高性能的流式数据处理功能，支持 DAG 任务图、窗口、 watermark 等高级特性。

### 3.1 核心类

#### Stream

流核心抽象类。

**构造函数：**
```python
from stream_processor import Stream

stream = Stream(name="my_stream", parallelism=1)
```

**主要方法：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `emit()` | record | Stream | 发射记录到流 |
| `emit_watermark()` | watermark | Stream | 发射 watermark |
| `emit_control()` | control_type, data | Stream | 发射控制记录 |
| `map()` | func | Stream | 映射转换 |
| `filter()` | predicate | Stream | 过滤 |
| `flat_map()` | func | Stream | 扁平映射 |
| `key_by()` | key_extractor | KeyedStream | 按键分组 |
| `close()` | - | None | 关闭流 |
| `clear()` | - | None | 清空流记录 |

**示例：**
```python
from stream_processor import Stream, Record

# 创建流
stream = Stream(name="data_stream")

# 发射数据
stream.emit(Record(value={"id": 1, "data": "hello"}))
stream.emit(Record(value={"id": 2, "data": "world"}))

# 转换操作
mapped = stream.map(lambda r: Record(value={**r.value, "processed": True}))
filtered = mapped.filter(lambda r: r.value["id"] > 1)

# 遍历
for record in filtered:
    print(record.value)
```

#### Record

数据记录。

```python
from stream_processor import Record
```

### 3.5 完整示例

```python

# 创建数据源
```

```python
"""
```

```python
解决方案：流式处理，支持窗口操作
"""

from stream_processor import Stream, Record, CollectionSource, TumblingEventTimeWindows
from stream_processor.operators import MapOperator, FilterOperator, TumblingWindowOperator

# 模拟日志数据
def generate_logs():
    import time
    logs = []
    for i in range(1000):
        logs.append(Record(
            value={
                'level': 'INFO' if i % 10 != 0 else 'ERROR',
                'message': f'Log message {i}',
                'service': f'service_{i % 5}',
                'timestamp': time.time() - (1000 - i)
            },
            timestamp=time.time() - (1000 - i)
        ))
    return logs

def process_log_stream():
    # 创建数据源
    logs = generate_logs()
    source = CollectionSource("logs", logs)
    
    # 创建流
    stream = Stream("log_stream")
    
    # 读取数据
    for log in source.read():
        stream.emit(log)
    
    # 过滤 ERROR 日志
    error_stream = stream.filter(lambda r: r.value['level'] == 'ERROR')
    
    # 按服务分组
    keyed = error_stream.key_by(lambda r: r.value['service'])
    
    # 窗口统计（每 10 秒窗口）
    window_op = TumblingWindowOperator(
        TumblingEventTimeWindows.of(size=10),
        reducer=lambda agg, r: agg + 1
    )
    
    # 处理结果
    print("服务错误统计:")
    for service, count in window_op.process(keyed):
        print(f"{service}: {count} 个错误")

    process_log_stream()
```

### 8. buffer_manager - 网络服务器场景

**场景：网络服务器缓冲区管理**

```python
"""
场景：网络服务器缓冲区管理
问题：并发连接多，缓冲区管理复杂
解决方案：使用缓冲池和无锁队列
"""

from buffer_manager import BufferPool, SPSCQueue
import threading
import time

def network_server_buffers():
    """网络服务器缓冲区管理"""
    # 创建缓冲池
    pool = BufferPool(buffer_size=4096, num_buffers=100)
    
    # 创建队列
    queue = SPSCQueue(capacity=1000)
    
    # 模拟客户端处理
    def client_handler():
        for i in range(100):
            # 获取缓冲区
            buffer = pool.acquire(timeout=1.0)
            if buffer:
                # 模拟数据处理
                buffer.write(f"Client data {i}".encode('utf-8'))
                
                # 放入队列
                queue.enqueue(buffer)
                
                time.sleep(0.01)
    
    # 模拟服务器处理
    processed = 0
    def server_processor():
        nonlocal processed
        while processed < 1000:
            buffer = queue.dequeue(timeout=1.0)
            if buffer:
                # 处理数据
                data = buffer.read()
                print(f"处理: {data.decode('utf-8')}")
                
                # 释放缓冲区
                pool.release(buffer)
                processed += 1
    
    # 启动线程
    client_threads = [threading.Thread(target=client_handler) for _ in range(10)]
    server_thread = threading.Thread(target=server_processor)
    
    for t in client_threads:
        t.start()
    server_thread.start()
    
    for t in client_threads:
        t.join()
    server_thread.join()
    
    # 统计
    stats = pool.get_stats()
    print(f"\n处理完成:")
    print(f"  总处理: {processed} 个请求")
    print(f"  缓冲池状态: 可用={stats.available}, 已用={stats.in_use}")

if __name__ == "__main__":
    network_server_buffers()
```

### C. 性能优化建议

1. **mem_mapper**：
   - 大文件（>2MB）启用大页
   - 顺序访问使用 `advise(region, 'sequential')`

   - 小块分配使用 SlabAllocator
   - 大块分配使用 BuddyAllocator
   - 实时系统使用 TLSFAllocator

   - 生产者-消费者场景使用 SPSCQueue
   - 多线程场景使用 MPMCQueue
   - 频繁访问使用 LRU 策略

   - 稀疏度 < 1% 使用 COO
   - 稀疏度 1-10% 使用 CSR
